# 1- MCP Native Static Viewer Refactor

## Amac

Bu isin amaci Blueprint MCP Server'i zorunlu HTTP runtime'dan ayirip kullanicinin kendi projesinde sade, local ve MCP-native bir deneyim sunmaktir. Yazilimci backend, frontend veya baska bir proje turunde bu MCP server'i kullandiginda ek bir HTTP server baslatmak zorunda kalmamalidir. Ana urun MCP tool'lari ve local Blueprint artifact'leri uretmeli; viewer ise bu artifact'leri tuketen opsiyonel bir cikti olmalidir.

Mevcut React/Tailwind frontend tasarimi korunacak, ancak veri kaynagi HTTP endpoint'lerinden embedded static viewer datasina tasinacaktir. `BlueprintExplorer`, `BlueprintCanvas` ve `GroupDetailPanel` gibi prop tabanli gorsel bilesenler korunabilir; esas refactor `BlueprintApp` data loading akisi, servis sinirlari ve renderer tarafinda yapilacaktir.

Blueprint memory kullanicinin proje kokunde acik `blueprint/` klasoru olarak degil, varsayilan olarak gizli `.blueprint/` klasoru altinda olusacaktir. MCP package icinde yalnizca sabit viewer bundle, template ve agent talimatlari bulunacak; kullanici projesine ozel analiz sonuclari package veya `node_modules` icine yazilmayacaktir.

Bu is kapsaminda zorunlu HTTP server, SSE live update ve runtime API akisi ana urun yolundan cikarilacaktir. Ilk hedef snapshot static HTML uretimi ve browser'da acma akisini dogru kurmaktir. Otomatik watch/live reload daha sonraki is olarak ele alinacaktir.

## Aldigimiz Kararlar

- Core urun MCP tool'lari ve local artifact uretimi olarak kalacak; kullanici zorunlu HTTP server baslatmayacak.
- Blueprint ciktilari kullanicinin proje kokunde varsayilan olarak `.blueprint/` altinda olusacak.
- MCP package icinde viewer bundle, shell template ve agent talimatlari bulunacak; proje-ozel Blueprint memory package icine yazilmayacak.
- Static viewer tek dosya olarak kullanicinin `.blueprint/index.html` dosyasina uretilecek.
- Viewer HTML self-contained olacak: inline JS, inline CSS ve guvenli embedded JSON data kullanilacak.
- Embedded data icin `window.__BLUEPRINT_DATA__` yerine `<script id="blueprint-data" type="application/json">` modeli kullanilacak.
- JSON HTML icine gomulmeden once XSS ve `</script>` kapanis risklerine karsi guvenli sekilde escape edilecek.
- Mevcut HTTP veri sekillendirme mantigi once transport-agnostic `BlueprintViewerService` benzeri bir servise tasinacak.
- Frontend `BlueprintDataSource` abstraction'i ile calisacak; static modda embedded data source, gecici/dev modda HTTP data source kullanilabilecek.
- `AGENTS.md` kullanicinin mevcut icerigi bozulmadan marker tabanli patch edilecek; dosya yoksa olusturulacak, varsa sadece Blueprint snippet'i eklenecek veya marker ici guncellenecek.
- `AGENTS.md` snippet'i package icindeki detayli kurallara yonlendirecek: `node_modules/blueprint-mcp/docs/agents.md`.
- `.gitignore` otomatik degistirilmeyecek; `.blueprint/` klasorunun commit edilip edilmeyecegi kullanici/ekip tercihine birakilacak.
- HTTP server hemen silinmeyecek; once service refactor, test tasima ve static renderer dogrulamasi tamamlanacak.
- SSE live update ilk asamada tasinmayacak; `blueprint watch` ve gercek live reload sonraki is olarak degerlendirilecek.
- Her gelistirme TDD yontemiyle yapilacak: once davranisi tarif eden test yazilacak, testin kirmizi oldugu gorulecek, sonra implementasyon yapilip test yesile cekilecek.
